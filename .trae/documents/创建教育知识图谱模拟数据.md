## 1. 数据生成方案

### 1.1 节点生成计划
- **学生节点 (Student)**：生成至少40个学生，包含不同姓名、学号、专业、年级等信息
- **教师节点 (Teacher)**：生成5-8个教师，覆盖不同学科
- **课程节点 (Course)**：生成10-15门课程，包含不同难度级别
- **知识点节点 (KnowledgePoint)**：生成20-30个知识点，覆盖不同课程内容
- **错误类型节点 (ErrorType)**：生成10-15种错误类型，包含不同严重程度

### 1.2 关系生成计划
- **TEACHES**：每位教师教授2-4门课程
- **LEARNS**：每位学生选修3-6门课程
- **CONTAINS**：每门课程包含2-5个知识点
- **HAS_ERROR**：每个知识点关联1-3种错误类型
- **CHAT_WITH**：学生之间随机生成聊天关系
- **LIKES**：学生之间随机生成点赞关系
- **RELATES_TO**：知识点之间生成关联关系

## 2. 数据生成脚本设计

### 2.1 技术选型
- 使用Python编写数据生成脚本
- 利用Neo4j Python驱动连接数据库
- 使用Faker库生成真实模拟数据
- 支持批量插入以提高性能

### 2.2 脚本结构
```python
# 1. 初始化数据库连接
# 2. 生成基础节点数据
# 3. 生成关系数据
# 4. 验证数据完整性
# 5. 关闭数据库连接
```

### 2.3 数据生成规则
- **学生属性**：生成合理的姓名、学号、年龄、性别分布
- **课程属性**：设置合理的难度级别（beginner/intermediate/advanced）
- **知识点属性**：设置合理的分类和描述
- **错误类型属性**：设置合理的严重程度（low/medium/high）
